题目简述：

> 你将得到一个整数数组 `matchsticks` ，其中 `matchsticks[i]` 是第 `i` 个火柴棒的长度。你要用 **所有的火柴棍** 拼成一个正方形。你 **不能折断** 任何一根火柴棒，但你可以把它们连在一起，而且每根火柴棒必须 **使用一次** 。
>
> 如果你能使这个正方形，则返回 `true` ，否则返回 `false` 。
>
> **提示:**
>
> - `1 <= matchsticks.length <= 15`
> - `1 <= matchsticks[i] <= 10^8`

题目链接：[473. 火柴拼正方形](https://leetcode.cn/problems/matchsticks-to-square/)

# 回溯算法

## 思路

看到这个题目的数据范围我就想到要考虑回溯算法了。

其中有许多细节，例如先排序然后按从大到小的顺序尝试拼接边，能够最大化算法效率。这些都是回溯算法的经典剪枝手法，这里就不详细说明了，请结合代码的注释。

## 代码

这是一个基本的回溯算法实现，针对不同火柴的组合进行搜索。

```java
class Solution {
    public boolean makesquare(int[] matchsticks) {
        long sum = 0;
        int maxMatchstick = 0;
        for (int matchstick : matchsticks) {
            sum += matchstick;
            maxMatchstick = Math.max(maxMatchstick, matchstick);
        }
        // 不能被 4 整除，不可能构成正方形
        if (sum % 4 != 0) return false;
        int sideLen = (int) (sum / 4);
        // 任意一根火柴比边长更长，也正方形
        if (maxMatchstick > sideLen) return false;

        // 倒序搜索，遍历的优先级为从大到小
        Arrays.sort(matchsticks);
        reverse(matchsticks);
        return dfs(matchsticks, new boolean[matchsticks.length], sideLen, 0, 0, 0);
    }

    public boolean dfs(int[] arr, boolean[] used, int sideLen, int validSide, int len, int i) {
        if (validSide == 4) return true;
        if (len == sideLen) return dfs(arr, used, sideLen, validSide + 1, 0, 0);

        for (int j = i; j < arr.length; j++) {
            if (!used[j] && len + arr[j] <= sideLen) {
                used[j] = true;
                if (dfs(arr, used, sideLen, validSide, len + arr[j], j + 1)) return true;
                used[j] = false;
            }
        }

        return false;
    }

    public void reverse(int[] arr) {
        for (int i = 0; i < arr.length / 2; i++) {
            int temp = arr[i];
            arr[i] = arr[arr.length - 1 - i];
            arr[arr.length - 1 - i] = temp;
        }
    }
}
```

# 状态压缩 DP

## 思路

上述的基本回溯算法存在一些重复搜索，算法效率还可以进一步优化。例如，当拥有可用火柴 $(2_{(1)},2_{(2)},4_{(3)})$ 时，上述回溯算法会分别尝试选取 $2_{(1)}$ 或 $2_{(2)}$ 拼成边，因为在算法看来他们是不同的火柴，所有的组合都需要被考虑——但实际上在只选择一个火柴时，二者选取谁是无关紧要的，是等价的，所以这是重复的尝试与回溯。

要避免这类重复搜索，就不能再针对不同火柴的组合进行搜索；若考虑状压 DP，则可以转为考虑针对拼接边的已积累长度进行搜索，这样就能从根本上避免这类重复搜索。

考虑状压 DP，将正方形的四条边分别编号并视为 “桶”，然后尝试向其中装入火柴。不再使用 `used` 数组记录火柴的使用信息，改为使用位掩码整数表示火柴的使用信息，并且定义这样的位掩码为状态，例如 $(01011)_2=(11)_{10}$ 表示共有五根火柴，且使用了其中第 2、4、5 根火柴。假设状态为 $x$，定义 $dp[x]$ 表示为火柴使用状态如掩码 $x$ 所示时当前正在拼凑的边的已积累长度（可以通过对 `sideLen` 取模来更优雅地实现），于是便不难得到状态转移。

## 代码

其中 `if (dp[state - (1 << i)] + matchsticks[n - i - 1] > sideLen) continue;` 我最初漏掉了，这会让算法不能按预期工作。比如，当已积累边长 3 而边长一共为 4，那么长度为 2 的火柴就不应该被考虑进当前正在拼接的边中，因为 $3+2>4$。

```java
class Solution {
    public boolean makesquare(int[] matchsticks) {
        int n = matchsticks.length;
        long sum = 0;
        int maxMatchstick = 0;
        for (int matchstick : matchsticks) {
            sum += matchstick;
            maxMatchstick = Math.max(maxMatchstick, matchstick);
        }
        if (sum % 4 != 0) return false;
        int sideLen = (int) (sum / 4);
        if (maxMatchstick > sideLen) return false;

        Arrays.sort(matchsticks);
        
        int N = 1 << n;
        int[] dp = new int[N];
        Arrays.fill(dp, -1);
        dp[0] = 0;

        for (int state = 1; state < N; state++) {
            for (int i = 0; i < n; i++) {
                /* 只要找到一个解即可，要求该位置上新状态为 1、旧状态为 0，从而得到状态转移 */
                if (((state >> i) & 1) == 1 && dp[state - (1 << i)] > -1) {
                    /* 已积累边长加上新火柴的长度不应超过边长！ */
                    if (dp[state - (1 << i)] + matchsticks[n - i - 1] > sideLen) continue;

                    dp[state] = (dp[state - (1 << i)] + matchsticks[n - i - 1]) % sideLen;
                    break;
                }
            }
        }

        return dp[N - 1] == 0;
    }
}
```

# 进一步剪枝？

上述两个算法是我实现的，他们都能 AC，但效率不是最理想的。我询问了 GPT 5.3，他给我了如下回答，仅供参考：

> 你的两个实现其实**算法思想没问题**，慢主要不是“大方向错了”，而是一些**搜索空间没有被压缩**的细节差异。这个题的性能差距往往来自 **对称状态剪枝（symmetry pruning）**。
>
> 核心结论先说：
>
> > 别人的回溯之所以更快，关键在这一句剪枝：
> > `if(i>0 && buckets[i] == buckets[i-1]) continue;`
> > 这一步消除了 **4! 级别的对称搜索**。
> > 你的两个实现都没有消除这种对称状态。
>
> 下面分别拆解。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">一、你的回溯算法为什么慢</span>
>
> 你的 DFS 实际上在做：
>
> - 先拼第一条边
> - 再拼第二条边
> - 再拼第三条边
> - 再拼第四条边
>
> 也就是说状态是：
>
> ```
> (validSide, len, used[])
> ```
>
> 问题在于：
>
> **四条边是完全等价的。**
>
> 例如：
>
> ```
> 边1 = {a,b}
> 边2 = {c,d}
> 边3 = {e}
> 边4 = {f}
> ```
>
> 和
>
> ```
> 边1 = {c,d}
> 边2 = {a,b}
> 边3 = {e}
> 边4 = {f}
> ```
>
> 是**同一个解**。
>
> 但你的 DFS 会把它们当作不同情况全部搜索。
>
> 所以搜索空间其实是：
>
> ```
> O(4! * 实际搜索)
> ```
>
> 因为边的顺序被重复枚举。
>
> ------
>
> <span style="font-weight: bold; font-size: 1.4em;">更大的问题</span>
>
> 你的代码里这一句：
>
> ```java
> if (len == sideLen) return dfs(arr, used, sideLen, validSide + 1, 0, 0);
> ```
>
> 意味着：
>
> 每条边都会 **重新从0开始遍历火柴**。
>
> 例如：
>
> ```
> 边1 可能用了 7,6
> 边2 仍然会从 7 开始尝试
> ```
>
> 虽然 `used` 会挡住，但仍然会做很多 **无意义遍历**。
>
> 典型重复：
>
> ```
> (边1) 7+6
> (边2) 5+4
> 
> 和
> 
> (边1) 5+4
> (边2) 7+6
> ```
>
> 被完整搜索两遍。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">二、别人回溯为什么更快</span>
>
> 他的策略完全不同：
>
> 不是“拼边”，而是：
>
> > **把每根火柴放进四个桶之一**
>
> 状态：
>
> ```
> buckets[4]
> ```
>
> 表示当前四条边长度。
>
> 递归：
>
> ```
> process(nums, idx, buckets)
> ```
>
> 每次处理一根火柴。
>
> ------
>
> 关键剪枝：
>
> ```java
> if(i>0 && buckets[i] == buckets[i-1]) continue;
> ```
>
> 这个剪枝非常关键。
>
> 例如：
>
> 当前
>
> ```
> buckets = [5,5,3,2]
> ```
>
> 当你尝试放火柴 `4`：
>
> ```
> 放 bucket0 -> [9,5,3,2]
> 放 bucket1 -> [5,9,3,2]
> ```
>
> 这两个状态其实是 **完全等价** 的：
>
> ```
> [9,5,3,2] ≡ [5,9,3,2]
> ```
>
> 因为桶没有顺序。
>
> 所以：
>
> ```
> 如果 buckets[i] == buckets[i-1]
> 跳过
> ```
>
> 这样就只会尝试一次。
>
> ------
>
> <span style="font-weight: bold; font-size: 1.4em;">剪枝效果</span>
>
> 假设有 4 个桶：
>
> 理论情况：
>
> ```
> 4^15 ≈ 1e9
> ```
>
> 但加了对称剪枝后：
>
> 实际搜索接近：
>
> ```
> 组合数级别
> ```
>
> 所以速度非常快。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">三、你的状压 DP 为什么也不快</span>
>
> 你的 DP：
>
> ```
> 状态数 = 2^n
> n ≤ 15
> ```
>
> 所以最多：
>
> ```
> 2^15 = 32768
> ```
>
> 每个状态枚举：
>
> ```
> n
> ```
>
> 理论复杂度：
>
> ```
> O(n * 2^n) ≈ 5e5
> ```
>
> 按理说应该很快。
>
> 但你这里有两个问题。
>
> ------
>
> <span style="font-weight: bold; font-size: 1.4em;">问题1：排序被破坏</span>
>
> 你排序了：
>
> ```java
> Arrays.sort(matchsticks);
> ```
>
> 但之后用了：
>
> ```java
> matchsticks[n - i - 1]
> ```
>
> 这导致：
>
> ```
> i 与火柴长度没有直接对应
> ```
>
> CPU cache locality 会变差，而且逻辑可读性下降。
>
> 应该直接：
>
> ```
> matchsticks[i]
> ```
>
> 并 **降序排序**。
>
> ------
>
> <span style="font-weight: bold; font-size: 1.4em;">问题2：DP仍然有大量无效状态</span>
>
> 你的 DP 会遍历：
>
> ```
> 所有 state
> ```
>
> 但很多 state 根本不可能形成合法边。
>
> 例如：
>
> ```
> state = {8,7}
> sideLen = 10
> ```
>
> 这种已经超边了。
>
> 虽然你有：
>
> ```
> > sideLen continue
> ```
>
> 但 **状态还是被枚举了**。
>
> 而 DFS 会在更早阶段剪掉。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">四、为什么 LeetCode 上最优解几乎全是 “桶回溯”</span>
>
> 这个问题其实有一个经典结论：
>
> > **4桶装填 + 对称剪枝 是最优搜索策略**
>
> 状态：
>
> ```
> buckets[4]
> ```
>
> 搜索树：
>
> ```
> 每根火柴 4 个选择
> ```
>
> 但由于剪枝：
>
> ```
> if(bucket[i]==bucket[i-1]) skip
> ```
>
> 等价状态被压缩。
>
> 实际复杂度非常低。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">五、你的回溯怎么改能快很多</span>
>
> 只要改一个思路：
>
> 不要“拼边”，改成“放桶”。
>
> 核心代码：
>
> ```java
> boolean dfs(int idx){
>     if(idx==-1) return true;
> 
>     for(int i=0;i<4;i++){
>         if(buckets[i]+nums[idx]>target) continue;
> 
>         if(i>0 && buckets[i]==buckets[i-1]) continue;
> 
>         buckets[i]+=nums[idx];
> 
>         if(dfs(idx-1)) return true;
> 
>         buckets[i]-=nums[idx];
>     }
> 
>     return false;
> }
> ```
>
> 复杂度会直接下降一个量级。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">六、还有一个你可能不知道的超级剪枝</span>
>
> 再加一个：
>
> ```
> 如果一个火柴放进空桶失败
> 就不用试其他空桶
> ```
>
> 写法：
>
> ```java
> for(int i=0;i<4;i++){
> 
>     if(buckets[i]+nums[idx]>target) continue;
> 
>     buckets[i]+=nums[idx];
> 
>     if(dfs(idx-1)) return true;
> 
>     buckets[i]-=nums[idx];
> 
>     if(buckets[i]==0) break;
> }
> ```
>
> 原因：
>
> ```
> 空桶之间完全等价
> ```
>
> 这个剪枝能再减少约 **4倍搜索**。
>
> ------
>
> <span style="font-weight: bold; font-size: 2.4em;">七、总结</span>
>
> 你慢的原因主要是三个：
>
> <span style="font-weight: bold; font-size: 1.4em;">1 回溯算法</span>
>
> 缺少 **桶对称剪枝**
>
> 导致：
>
> ```
> 4! 倍重复搜索
> ```
>
> ------
>
> <span style="font-weight: bold; font-size: 1.4em;">2 回溯建边策略</span>
>
> ```
> 先拼边
> ```
>
> 导致：
>
> ```
> 边顺序重复
> ```
>
> ------
>
> <span style="font-weight: bold; font-size: 1.4em;">3 状压 DP</span>
>
> 虽然理论复杂度低，但：
>
> - 仍遍历全部 `2^n`
> - 剪枝较晚
> - 常数较大
>
> ------
>
> ✔ **最优策略**
>
> ```
> 降序排序
> + 桶回溯
> + 对称剪枝
> + 空桶剪枝
> ```
>
> 这是这个题的公认最优解。
>
