题目简述：

> 给你一个字符串 `s` 和一个整数 `k` 。你可以选择字符串中的任一字符，并将其更改为任何其他大写英文字符。该操作最多可执行 `k` 次。
>
> 在执行上述操作后，返回 *包含相同字母的最长子字符串的长度。*

题目链接：[424. 替换后的最长重复字符](https://leetcode.cn/problems/longest-repeating-character-replacement/)

# 思路

很明显的滑动窗口问题。将本问题的问题描述换个说法，就是要求滑动窗口内除频数最高的字符以外之字符，其频数总和不能高于 `k`。这种情况下，让滑动窗口的右端点遍历每一个可能的位置，即固定窗口右端点，然后收缩窗口左端点，直到满足窗口内除频数最高的字符外所有字符频数总和不大于 `k`。

对每个固定的窗口右端点，我们都得到尽可能长的合法窗口（尽可能地让除频数最高的字符外所有字符频数恰等于 `k`）；这样，当窗口右端点遍历所有点后，我们就能得到最长的合法窗口，即包含经过替换后含有相同字母的最长子字符串。

# 代码

算法的时间复杂度为 $O(|\Sigma|n)$，空间复杂度为 $O(|\Sigma|)$，其中 $|\Sigma|$ 为字符集包含的元素数量。

对于本问题，$|\Sigma|$ 为一个很小的常数，因此渐进时间复杂度为 $O(n)$，空间开销为 $O(1)$。

> 其实可以不实时更新 `maxFreq` 和 `maxFreqChar`，而只在 `j` 增大时更新 `maxFreq`，便也就不需要 `maxFreqChar` 了——可以被完全移除。这样的话，时间复杂度就可以严谨地与 $|\Sigma|$ 无关了，代码也能更简洁。不过，我们下方的代码实现里依然是考虑实时更新的，这更符合直觉。
>
> 若不实时更新 `maxFreq`，则 `maxFreq` 的含义不再是实时的窗口内频率最高字符之频数，而是窗口的历史频率最高字符之频数，在窗口收缩时无需重新计算。读者可以思考一下为什么这样也是可行的。

```java
class Solution {
    public int characterReplacement(String s, int k) {
        int i = 0;
        int[] freq = new int[26];
        int maxFreq = 0;
        char maxFreqChar = '\0';
        int res = 0;

        for (int j = 0; j < s.length(); j++) {
            char c = s.charAt(j);
            freq[c - 'A']++;

            if (freq[c - 'A'] >= maxFreq) {
                maxFreq = freq[c - 'A'];
                maxFreqChar = c;
            }

            int diff = j - i + 1 - maxFreq;
            while (diff > k) {
                char cc = s.charAt(i++);
                freq[cc - 'A']--;
                if (cc == maxFreqChar) {
                    maxFreq = 0;
                    for (int t = 0; t < 26; t++) {
                        if (freq[t] > maxFreq) {
                            maxFreq = freq[t];
                            maxFreqChar = (char) (t + 'A');
                        }
                    }
                }
                diff = j - i + 1 - maxFreq;
            }

            res = Math.max(res, j - i + 1);
        }

        return res;
    }
}
```
