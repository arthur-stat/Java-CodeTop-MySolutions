题目简述：

> 给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** *`target`* 的那 **两个** 整数，并返回它们的数组下标。
>
> 你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。
>
> 你可以按任意顺序返回答案。

题目链接：[两数之和](https://leetcode.cn/problems/two-sum/description/)

# 思路

我们当然可以直接遍历扫描一下，这是最暴力的方案：

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int n = nums.length - 1;
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n + 1; j++) {
                if (nums[i] + nums[j] == target) return new int[] {i, j};
            }
        }
        return new int[0];
    }
}
```

但这样的时间复杂度稍高，因为我们嵌套了双重循环，时间复杂度为 $O(n^2)$。为降低时间开销，我们可以利用哈希表提升算法的效率。

具体来说，因为 $target$ 是确定的，那么对于遍历过程中访问的元素 $nums[i]$，我们只需要知道两件事就可以确认 $nums[i]$ 是否可能与其他元素构成解：

1. $target-nums[i]$ 是否位于数组中；
2. 如果存在，则（其中一个）$target-nums[i]$ 位于数组中的索引是什么。

这显然很适合用哈希表来解决。我们只需要进行一次遍历扫描，在扫描的过程中使用哈希表记录已扫描元素的值到索引的映射（哈希表的键为元素的值、哈希表的值为元素的索引），然后对于当前访问的元素 $nums[i]$，在哈希表中查询 $target-nums[i]$ 是否作为键被存储在哈希表中。如果是，则我们就找出了一组解：哈希表中 $target-nums[i]$ 作为键对应的索引值以及当前元素的索引值。

# 代码

算法的时间复杂度为 $O(n)$，空间复杂度为 $O(n)$，是用空间换时间的做法。

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 1; i < nums.length; i++) {
            map.put(nums[i-1], i-1);
            if (map.containsKey(target - nums[i])) return new int[]{map.get(target - nums[i]), i};
        }
        return new int[0];
    }
}
```
