题目简述：

> 请设计一个自助结账系统，该系统需要通过一个队列来模拟顾客通过购物车的结算过程，需要实现的功能有：
>
> - `get_max()`：获取结算商品中的最高价格，如果队列为空，则返回 -1
> - `add(value)`：将价格为 `value` 的商品加入待结算商品队列的尾部
> - `remove()`：移除第一个待结算的商品价格，如果队列为空，则返回 -1
>
> 注意，为保证该系统运转高效性，以上函数的均摊时间复杂度均为 O(1)

题目链接：[LCR 184. 设计自助结算系统](https://leetcode.cn/problems/dui-lie-de-zui-da-zhi-lcof/)

# 思路

本问题与 [859 · 最大栈](https://www.lintcode.com/problem/859/) 是相似的，实际上是要求我们设计「最大队列」数据结构。

与 [859 · 最大栈](https://www.lintcode.com/problem/859/) 的双栈解决思路不同，我们不实现相似的「双队列」，我的解决方案是转变思路。实际上，针对队列有一种特别的视角是一段数组上的滑动窗口：当执行 `offer` 时，等价于窗口右端点递增；当执行 `poll` 时，等价输出窗口左端点的值，然后窗口左端点递增。显然，窗口内的元素就是队列内的元素。显然这符合 FIFO 结构，因此这样解释队列也是合理的。

要高效地找出队列的最大值听起来略有难度，但转变为滑动窗口的思路后，问题就变成了**维护滑动窗口的最大值**——而这是**单调队列**的模板问题。关于单调队列的理论，可以参考我为 [239. 滑动窗口最大值](https://leetcode.cn/problems/sliding-window-maximum/) 撰写的题解。

因此，本问题实际上是实现单调队列数据结构。

# 代码

该数据结构的各操作均摊时间复杂度为 $O(1)$。

```java
class Checkout {

    private Deque<Integer> q;  // FIFO 队列
    private Deque<Integer> p;  // 单调队列

    public Checkout() {
        q = new ArrayDeque<>();
        p = new ArrayDeque<>();
    }
    
    public int get_max() {
        return q.isEmpty() ? -1 : p.peekFirst();
    }
    
    public void add(int value) {
        q.offer(value);
        while (!p.isEmpty() && p.peekLast() < value) {
            p.removeLast();
        }
        p.addLast(value);
    }
    
    public int remove() {
        if (q.isEmpty()) return -1;
        int res = q.poll();
        if (p.peekFirst() == res) {
            p.removeFirst();
        }
        return res;
    }
}

```
