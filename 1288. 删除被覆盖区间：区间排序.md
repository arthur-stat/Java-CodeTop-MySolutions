题目简述：

> 给你一个区间列表，请你删除列表中被其他区间所覆盖的区间。
>
> 只有当 `c <= a` 且 `b <= d` 时，我们才认为区间 `[a,b)` 被区间 `[c,d)` 覆盖。
>
> 在完成所有删除操作后，请你返回列表中剩余区间的数目。

题目链接：[1288. 删除被覆盖区间](https://leetcode.cn/problems/remove-covered-intervals/)

# 思路

典型的区间排序问题。针对本问题，按区间左端点排序后，根据右端点判断区间是否能被覆盖。

# 代码

算法的时间复杂度为 $O(n\log n)$，空间复杂度为 $O(\log n)$——这些都是排序算法带来的开销。

这里有一个细节是<u>当两个区间的左端点相等时我们按右端点的倒序排序</u>，这是为了保证除非两个区间完全相同，否则排序后只可能是右侧的区间能够包含左侧的区间，这样就能确保「排序后只需要观察右端点就能判断区间能否被覆盖」成立。

```java
class Solution {
    public int removeCoveredIntervals(int[][] intervals) {
        Arrays.sort(intervals, (a, b) -> {
            return (a[0] != b[0]) ? Integer.compare(a[0], b[0]) : Integer.compare(b[1], a[1]);
        });

        int count = 1;
        int l = intervals[0][0];
        int r = intervals[0][1];

        for (int i = 1; i < intervals.length; i++) {
            if (intervals[i][1] > r) {
                count++;
                l = intervals[i][0];
                r = intervals[i][1];
            }
        }

        return count;
    }
}
```
