题目简述：

> 在一个火车旅行很受欢迎的国度，你提前一年计划了一些火车旅行。在接下来的一年里，你要旅行的日子将以一个名为 `days` 的数组给出。每一项是一个从 `1` 到 `365` 的整数。
>
> 火车票有 **三种不同的销售方式** ：
>
> - 一张 **为期一天** 的通行证售价为 `costs[0]` 美元；
> - 一张 **为期七天** 的通行证售价为 `costs[1]` 美元；
> - 一张 **为期三十天** 的通行证售价为 `costs[2]` 美元。
>
> 通行证允许数天无限制的旅行。 例如，如果我们在第 `2` 天获得一张 **为期 7 天** 的通行证，那么我们可以连着旅行 7 天：第 `2` 天、第 `3` 天、第 `4` 天、第 `5` 天、第 `6` 天、第 `7` 天和第 `8` 天。
>
> 返回 *你想要完成在给定的列表 `days` 中列出的每一天的旅行所需要的最低消费* 。

题目链接：[983. 最低票价](https://leetcode.cn/problems/minimum-cost-for-tickets/)

# 暴力搜索

虽然本问题从优化角度看也符合「最小化目标单调函数」的描述，但如果固定总消费，是很难给出确定的组合的，因为此处目标函数不是常见的贪心计数函数，火车票面额有且仅有三类，因此不能套用「二分答案」的思路。

如果没有很直观清晰的思路，可以考虑暴力 DFS 搜索。至少，我起初是没有想到任何办法的，只好先实现记忆化搜索再一步步根据记忆化搜索导出了动态规划。

```java
class Solution {
    public int mincostTickets(int[] days, int[] costs) {
        return dfs(days, costs, days[0], 0);
    }

    /**
     * 暴力 DFS
     *
     * @param days 出行日期数组
     * @param costs 票价数组
     * @param today 今日（某个购票日）日期
     * @param totalCost 上一次购票后的总开销
     * @return 购票最小开销
     */
    public int dfs(int[] days, int[] costs, int today, int totalCost) {
        if (today > days[days.length - 1]) return totalCost;

        int res = Integer.MAX_VALUE;
        res = Math.min(res, dfs(days, costs, binarySearch(days, today + 1), totalCost + costs[0]));
        res = Math.min(res, dfs(days, costs, binarySearch(days, today + 7), totalCost + costs[1]));
        res = Math.min(res, dfs(days, costs, binarySearch(days, today + 30), totalCost + costs[2]));

        return res;
    }

    /**
     * 二分查找，返回首个大于 target 的 arr 数组内的值
     *
     * @param arr 输入数组
     * @param target 目标值
     * @return 输入数组内大于目标值的首个值
     */
    public int binarySearch(int[] arr, int target) {
        int lo = 0;
        int hi = arr.length - 1;
        while (lo <= hi) {
            int mid = lo + ((hi - lo) >>> 1);
            if (arr[mid] < target) {
                lo = mid + 1;
            } else {
                hi = mid - 1;
            }
        }
        return (lo > arr.length - 1) ? target : arr[lo];
    }
}
```

# 记忆化搜索

暴力搜索的算法可以进行改进，注意到若不考虑历史购票开销而是直接从某日开始定制出行计划，则必然存在唯一的最小购票开销，因此可以考虑记忆化搜索。

```java
class Solution {
    public int mincostTickets(int[] days, int[] costs) {
        return dfs(days, costs, new int[366], days[0], 0);
    }

    /**
     * 暴力 DFS
     *
     * @param days 出行日期数组
     * @param costs 票价数组
     * @param cache 缓存 today 起的最小购票开销
     * @param today 今日（某个购票日）日期
     * @param totalCost 上一次购票后的总开销
     * @return 购票最小开销
     */
    public int dfs(int[] days, int[] costs, int[] cache, int today, int totalCost) {
        if (today > days[days.length - 1]) return totalCost;
        if (cache[today] > 0) return totalCost + cache[today];

        int res = Integer.MAX_VALUE;
        res = Math.min(res, dfs(days, costs, cache, binarySearch(days, today + 1), totalCost + costs[0]));
        res = Math.min(res, dfs(days, costs, cache, binarySearch(days, today + 7), totalCost + costs[1]));
        res = Math.min(res, dfs(days, costs, cache, binarySearch(days, today + 30), totalCost + costs[2]));

        cache[today] = res - totalCost;
        return res;
    }

    /**
     * 二分查找，返回首个大于 target 的 arr 数组内的值
     *
     * @param arr 输入数组
     * @param target 目标值
     * @return 输入数组内大于目标值的首个值
     */
    public int binarySearch(int[] arr, int target) {
        int lo = 0;
        int hi = arr.length - 1;
        while (lo <= hi) {
            int mid = lo + ((hi - lo) >>> 1);
            if (arr[mid] < target) {
                lo = mid + 1;
            } else {
                hi = mid - 1;
            }
        }
        return (lo > arr.length - 1) ? target : arr[lo];
    }
}
```

算法 AC，并且击败了大部分提交的代码。

> 实际上，上文中算法所实现的二分查找是不必要的。之所以考虑二分查找，是因为输入参数是具体的日期，而非 `days` 的索引。但我们可以证明，下一个购票日必然是 `days` 内的某个日期，只有这样才有可能取得最优解。因此，我们可以改日期参数为日期数组索引，这样就不再需要二分查找以取得下一个购票日了，而可以直接从当前索引起线性递增遍历日期数组来搜索下一个购票日。

# 动态规划

在记忆化搜索的分析中有提到，「若不考虑历史购票开销而是直接从某日开始定制出行计划，则必然存在唯一的最小购票开销」，此处实际上是<u>**无后效性**</u>的体现，暗示我们本问题或许可以考虑动态规划。事实上，既然可以靠记忆化搜索解决，那么几乎有对应的动态规划解法。

参考 `dfs(today)` 的定义，设计 $dp[i]$ 表示自第 $1$ 天起至第 $i$ 天满足第 $i$ 天及其之前出行计划的最小购票开销，则状态转移方程为
$$
dp[i]=\left\{\begin{align}&dp[i-1],&&i\notin days\\&\min\big\{dp[i-1]+costs[0],\ dp[i-7]+costs[1],\ dp[i-30]+costs[2]\big\},&&i\in days\end{align}\right.
$$
最终 $dp\big[days[n]\big]$ 即为结果。

```java
class Solution {
    public int mincostTickets(int[] days, int[] costs) {
        int n = days.length;
        Set<Integer> set = Arrays.stream(days).boxed().collect(Collectors.toSet());
        int[] dp = new int[days[n - 1] + 1];
        for (int i = 1; i <= days[n - 1]; i++) {
            if (set.contains(i)) {
                int dp_0 = dp[i - 1] + costs[0];
                int dp_1 = (i > 7) ? dp[i - 7] + costs[1] : costs[1];
                int dp_2 = (i > 30) ? dp[i - 30] + costs[2] : costs[2];
                dp[i] = Math.min(dp_0, Math.min(dp_1, dp_2));
            } else {
                dp[i] = dp[i - 1];
            }
        }
        return dp[days[n-1]];
    }
}
```

> 动态规划算法也可以考虑按索引而不是实际日期实现，这样做会更稳健。
