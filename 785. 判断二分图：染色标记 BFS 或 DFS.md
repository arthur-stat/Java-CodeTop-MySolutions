题目简述：

> 存在一个 **无向图** ，图中有 `n` 个节点。其中每个节点都有一个介于 `0` 到 `n - 1` 之间的唯一编号。给你一个二维数组 `graph` ，其中 `graph[u]` 是一个节点数组，由节点 `u` 的邻接节点组成。形式上，对于 `graph[u]` 中的每个 `v` ，都存在一条位于节点 `u` 和节点 `v` 之间的无向边。该无向图同时具有以下属性：
>
> - 不存在自环（`graph[u]` 不包含 `u`）。
> - 不存在平行边（`graph[u]` 不包含重复值）。
> - 如果 `v` 在 `graph[u]` 内，那么 `u` 也应该在 `graph[v]` 内（该图是无向图）
> - 这个图可能不是连通图，也就是说两个节点 `u` 和 `v` 之间可能不存在一条连通彼此的路径。
>
> **二分图** 定义：如果能将一个图的节点集合分割成两个独立的子集 `A` 和 `B` ，并使图中的每一条边的两个节点一个来自 `A` 集合，一个来自 `B` 集合，就将这个图称为 **二分图** 。
>
> 如果图是二分图，返回 `true` ；否则，返回 `false` 。

题目链接：[785. 判断二分图](https://leetcode.cn/problems/is-graph-bipartite/)

# 暴力回溯枚举？完蛋！

起初我考虑的是类似模拟的办法，暴力回溯所有可能性。考虑两个分类集合，遍历把每个节点放入其中某个集合的可能性。某种意义上，这也是 DFS + 基本的剪枝。

不出意外地算法超时了。不过，假设我们有无限的时间，算法还是能正确求解问题的。

```java
class Solution {
    public boolean isBipartite(int[][] graph) {
        Set<Integer> A = new HashSet<>();
        Set<Integer> B = new HashSet<>();
        return dfs(graph, A, B, 0);
    }

    private boolean dfs(int[][] graph, Set<Integer> A, Set<Integer> B, int i) {
        if (i == graph.length) return true;

        boolean a = A.contains(i);
        boolean b = B.contains(i);

        if (a & !b) {
            if (!Arrays.stream(graph[i]).anyMatch(A::contains)) {
                Set<Integer> tmp = new HashSet<>(B);
                Arrays.stream(graph[i]).forEach(tmp::add);
                if (dfs(graph, A, tmp, i + 1)) return true;
            }
        } else if (!a & b) {
            if (!Arrays.stream(graph[i]).anyMatch(B::contains)) {
                Set<Integer> tmp = new HashSet<>(A);
                Arrays.stream(graph[i]).forEach(tmp::add);
                if (dfs(graph, tmp, B, i + 1)) return true;
            }
        } else if (!a & !b) {
            B.add(i);
            if (!Arrays.stream(graph[i]).anyMatch(B::contains)) {
                Set<Integer> tmp = new HashSet<>(A);
                Arrays.stream(graph[i]).forEach(tmp::add);
                if (dfs(graph, tmp, B, i + 1)) return true;
            }
            B.remove(i);

            A.add(i);
            if (!Arrays.stream(graph[i]).anyMatch(A::contains)) {
                Set<Integer> tmp = new HashSet<>(B);
                Arrays.stream(graph[i]).forEach(tmp::add);
                if (dfs(graph, A, tmp, i + 1)) return true;
            }
            A.remove(i);
        }

        return false;
    }
}
```

# BFS 或 DFS 染色

本问题的正解是 BFS 或 DFS，只需要对所有节点共进行一次线性时间复杂度的遍历，就能得到答案。具体而言，我们根本不需要遍历把每个节点放入某个集合的可能性，只需要保证遍历过程中的「染色」不矛盾即可。

以 BFS 为例，我们为一个节点染为红色，然后根据邻接表，将其邻接节点染为绿色。接下来我们不需要逐个考虑邻接表索引上的下一个节点是什么色，我们只需要把上一个节点的绿色邻接节点们都加入到队列，然后进行下一轮 BFS，并且将队列的输出视为下一个节点。这样，在完成一次遍历后，如果没有冲突，则可以判定该图是二分的。如果存在冲突，则在遍历的过程中就能发现了。

这样，我们就把有无数个状态分支、需要不断回溯的搜索树，简化为了状态数与图节点数相同、无回溯的搜索树。

> 以 BFS 为例，体现在编码上，就是把 `for i in graph` 遍历的主循环，改为了 `while len(q) > 0` 遍历的主循环。
>
> 不过，考虑到非连通图的可能，我们实际上还是需要进行 `for i in graph` 遍历，但该遍历的目的是找出非联通节点，而不再是按邻接表索引遍历。

实际上，图的二分是一个经典问题。这里有一个重要结论：**一个无向图是二分图，当且仅当它不包含奇数长度的环。**

嗯，没有学习过图论的话，可能较难察觉到其中的关系。总之，这是一个非常重要、经典且优雅的基本定理。

```java
class Solution {
    public boolean isBipartite(int[][] graph) {
        int n = graph.length;
        int[] color = new int[n];
        Deque<Integer> q = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            if (color[i] == 0) {
                // init
                color[i] = 1;
                q.offer(i);
                int num = 1;

                while (!q.isEmpty()) {
                    int j = q.poll();
                    num--;

                    for (int k : graph[j]) {
                        if (color[k] == color[j]) return false;
                        if (color[k] == 0) {
                            q.offer(k);
                            color[k] = -color[j];
                        }
                    }

                    if (num == 0) num = q.size();
                }
            }
        }

        return true;
    }
}
```
