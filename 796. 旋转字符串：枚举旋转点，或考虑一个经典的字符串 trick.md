题目简述：

> 给定两个字符串, `s` 和 `goal`。如果在若干次旋转操作之后，`s` 能变成 `goal` ，那么返回 `true` 。
>
> `s` 的 **旋转操作** 就是将 `s` 最左边的字符移动到最右边。 
>
> - 例如, 若 `s = 'abcde'`，在旋转一次之后结果就是`'bcdea'` 。

题目链接：[796. 旋转字符串](https://leetcode.cn/problems/rotate-string/)

# 常规思路：枚举旋转点

## 思路

在 [33. 搜索旋转排序数组](https://leetcode.cn/problems/search-in-rotated-sorted-array/) 的题解中，我有给出过这样一个结论：

> 无论旋转几次，一个旋转数组里最多只有 2 段递增序列，所以代码甚至原封不到就可以适用多次旋转的旋转数组。
>
> 从数学上可以证明：旋转数组无论旋转多少次，都与仅旋转一次是同构的。我在 [153. 寻找旋转排序数组中的最小值](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/) 的题解中给出了证明该结论的论文，而且针对那里数组非严格单调递增的情况结论依然适用，读者感兴趣的话可以去那篇题解看看。

因此，无论字符串的旋转发生过几次，我们都等价地视为一次即可。

这让我想到枚举旋转点考虑字符串哈希，这是一种效率极高的方案。

但简便起见，枚举旋转点对比拼接字符串是否与原串相等就可以了。

## 代码

算法的时间复杂度为 $O(n^2)$，空间复杂度为 $O(n)$。

如果考虑字符串哈希（滚动哈希）判等而不是 `equals()` 严格判等，则时间复杂度可以降至 $O(n)$。

```java
class Solution {
    public boolean rotateString(String s, String goal) {
        if (s.equals(goal)) return true;
        if (s.length() != goal.length()) return false;

        for (int i = 0; i < s.length() - 1; i++) {
            String tmp = s.substring(i + 1, s.length()) + s.substring(0, i + 1);
            if (goal.equals(tmp)) return true;
        }

        return false;
    }
}
```

# 更漂亮的思路：经典 trick

## 思路

其实直接 `return s.length() == goal.length() && (s + s).contains(goal);` 就能解决问题了，而且这是时间复杂度为 $O(n)$ 的优秀方案。

为什么呢？

这是一个经典的 trick，利用了一个非常常见的字符串命题：

> **<u>当 `goal` 与 `s` 长度相等时，`goal` 是 `s` 的旋转的充要条件为 `goal` 是 `s+s` 的子串。没有任何例外情况。</u>**

读者如果对此不熟悉，不妨试着推导一下。

## 代码

算法的时间复杂度为 $O(n)$，空间复杂度为 $O(n)$。

```java
class Solution {
    public boolean rotateString(String s, String goal) {
        return s.length() == goal.length() && (s + s).contains(goal);
    }
}
```
