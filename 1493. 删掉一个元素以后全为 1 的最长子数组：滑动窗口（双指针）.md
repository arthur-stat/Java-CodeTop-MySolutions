题目简述：

> 给你一个二进制数组 `nums` ，你需要从中删掉一个元素。
>
> 请你在删掉元素的结果数组中，返回最长的且只包含 1 的非空子数组的长度。
>
> 如果不存在这样的子数组，请返回 0 。

题目链接：[1493. 删掉一个元素以后全为 1 的最长子数组](https://leetcode.cn/problems/longest-subarray-of-1s-after-deleting-one-element/)

# 我的方案

## 思路

除开数组内全为 1 的情况，我们要找的是符合下方模式的最长子数组长度：
$$
\underbrace{111\cdots11}_{\text{left}}\ 0\ \underbrace{111\cdots11}_{\text{right}}
$$
即，left 和 right 两部分均为连续的 1，二者之间存在一个 0。

我们遍历计数，假设 left 计数是已知的（初始化计数为 0），假设当前遍历访问到 1，则增加 right 计数；假设当前遍历访问到 0，则尝试移除这个 0，体现在算法上就是将 right 的计数赋给 left，然后将 right 的计数置 0。这样，我们就可以找出所有符合上文模式的最长子数组长度。这里实际上存在某种单调性——实际上这个方案就是滑动窗口的变形，但思路更直白。

由于题目要求必须删除一个元素，所以要特别处理全为 1 的数组，因为上述算法流程只负责找到符合上文模式的最长子数组，不能处理全为 1 的数组。

## 代码

算法的时间复杂度为 $O(n)$，空间复杂度为 $O(1)$。

```java
class Solution {
    public int longestSubarray(int[] nums) {
        int l = 0;
        int r = 0;
        int res = 0;
        boolean nonezero = true;
        for (int num : nums) {
            if (num == 1) {
                r++;
            } else {
                l = r;
                r = 0;
                nonezero = false;
            }
            res = Math.max(res, l + r);
        }
        return nonezero ? nums.length - 1 : res;
    }
}
```

## 滑动窗口（双指针）

实际上我的方案可以被视为一个非典型实现的滑动窗口，将 left、0 和 right 视为统一的窗口即可，或考虑 left 左侧置左指针、right 右侧置右指针，则左右指针均单调不减。

逻辑上没有不同，可以参考[滑动窗口（Python/Java/C++/C/Go/JS/Rust）](https://leetcode.cn/problems/longest-subarray-of-1s-after-deleting-one-element/)。
