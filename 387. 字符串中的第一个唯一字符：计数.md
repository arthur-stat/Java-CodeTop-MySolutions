题目简述：

> 给定一个字符串 `s` ，找到 *它的第一个不重复的字符，并返回它的索引* 。如果不存在，则返回 `-1` 。

题目链接：[387. 字符串中的第一个唯一字符](https://leetcode.cn/problems/first-unique-character-in-a-string/)

# 两遍扫描

## 思路

这个方案十分直接，暴力地进行两次扫描，第一次扫描时记录词频，并第二次扫描时判断字符频数是否为 1。

这个方案会对字符串进行两次扫描，是因为在第一次扫描做词频统计时，这完全没有利用上索引的信息，便不可避免地需要进行第二次扫描。

## 代码

记字符集大小为 $|\Sigma|$，算法的时间复杂度为 $O(n)$（两次扫描），空间复杂度为 $O(|\Sigma|)$。

```java
class Solution {
    public int firstUniqChar(String s) {
        int[] freq = new int[26];
        for (char c : s.toCharArray()) freq[c - 'a']++;
        int res = Integer.MAX_VALUE;
        for (int i = 0; i < s.length(); i++) {
            if (freq[s.charAt(i) - 'a'] == 1) res = Math.min(res, i);
        }
        return (res == Integer.MAX_VALUE) ? -1 : res;
    }
}
```

# 哈希表 + 简单状态

## 思路

用哈希表记录已存在的字符即可——一个简单的哈希表计数问题。

扫描的过程中，对于当前扫描到的字符，考虑两种状态：

1. 首次扫描到该类字符；
2. 二次或多次扫描到字符。

于是，对于每种字符，就有了三种状态：

1. 被扫描 0 次；
2. 被扫描 1 次；
3. 被扫描 2 次及以上。

针对被扫描 1 次的字符记录其索引，最后判断字符的状态即可。

这是一个很简单的状态模型，该算法对字符串仅进行一遍扫描，与第一种方案对比主要是在第一遍扫描时就记录了索引，避免了二次扫描。

> 这是单哈希表的方案。也可以用两个哈希表做到这一点：一个哈希表照旧负责统计频数，另一个哈希表则负责记录字符首次出现的位置。
>
> 这个方案就不实现了，其实和其他方法没有本质上的区别。

## 代码

记字符集大小为 $|\Sigma|$，算法的时间复杂度为 $O(n)$（一次扫描），空间复杂度为 $O(|\Sigma|)$。

```java
class Solution {
    public int firstUniqChar(String s) {
        int[] exist = new int[26];
        /* 规定三种状态：-2 表示未出现；-1 表示出现多次；非负整数表示仅出现一次的位置 */
        Arrays.fill(exist, -2);
        for (int i = 0; i < s.length(); i++) {
            int j = s.charAt(i) - 'a';
            if (exist[j] == -2) exist[j] = i;
            else if (exist[j] >= 0) exist[j] = -1;
        }

        int res = Integer.MAX_VALUE;
        for (int i = 0; i < exist.length; i++) {
            if (exist[i] >= 0) res = Math.min(res, exist[i]);
        }
        return (res == Integer.MAX_VALUE) ? -1 : res;
    }
}
```
