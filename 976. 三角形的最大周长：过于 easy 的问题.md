题目简述：

> 给定由一些正数（代表长度）组成的数组 `nums` ，返回*由其中三个长度组成的、**面积不为零**的三角形的最大周长*。如果不能形成任何面积不为零的三角形，返回 `0`。

题目链接：[976. 三角形的最大周长](https://leetcode.cn/problems/largest-perimeter-triangle/)

# 思路

众所周知，三条边构成三角形的充要条件是任意两边之和大于第三边。另一个等价描述是，三条边中较短的两条边大于最长的一条边，当且仅当在三条边能够构成三角形时成立。

因此，对于排序后的三个数，只要最小的两个数之和大于最大的数，那么这三个数就能作为某个三角形的边长。

更进一步地，对于排序后的数组，我们固定一个数 $c$ 作为上限、只讨论小于该数的数，那么假设 $a,b$ 能够与 $c$ 共同作为边长构成三角形，即 $a+b>c$，如果存在 $d$ 使得 $d>a$，则 $d,b,c$ 也一定能作为边长构成三角形，因为 $d+b>a+b>c$。所以，当固定了一个数作为三角形的最长边时，周长最大的三角形一定由该数与是恰小于该数的相邻两个数作为边长构成。这里存在单调性。

因此，理论上我们只需要遍历所有数分别讨论他们作为最长边 $c$ 的情况，就能得到周长最长三角形的周长。遍历时，应针对排序后的数组倒序扫描，这是因为数组是已排序的，这里又存在一个单调性。综合两条单调性，可以保证扫描过程中首个满足条件 $a+b>c$ 的相邻三元数组 $(a,b,c)$ 就分别是周长最长三角形的三条边的长（故而我们不必完整遍历所有三元数组，可以提前返回结果）。

# 代码

算法的时间复杂度为 $O(n\log n)$，在不考虑排序算法带来的空间开销情况下，空间复杂度为 $O(1)$。

```java
class Solution {
    public int largestPerimeter(int[] nums) {
        Arrays.sort(nums);
        int n = nums.length;
        for (int i = n - 1; i > 1; i--) {
            if (nums[i - 2] + nums[i - 1] > nums[i]) return nums[i - 2] + nums[i - 1] + nums[i];
        }
        return 0;
    }
}
```
